  • Simulate an automated garage door system using a PLC and real components like buttons, motors, and LEDs.
  • Replicate garage door behavior through visual and mechanical feedback without using a real door.
  • Track usage cycles and simulate a maintenance lockout after repeated use.
  • Include emergency stop, obstruction detection, and manual override functionality for safety logic testing
  • System Planning and Setup
    • Designed a simulated garage system using a PLC to mirror real-world garage behavior.
    • User inputs included two keypads and a remote FOB, represented by push buttons:
      • Green button for "open"
      • Red button for "close"
      • Round red button for "stop"
    • Garage door motion was simulated using DC motors and LEDs to represent opening, closing, and system status.
    • System was programmed to lock out after 8 full open/close cycles, simulating a maintenance condition with a red warning light.
  • PLC and Hardware
    • Used Allen Bradley Compact GuardLogix 5380 PLC (5069-L306ERMS2) with 16-point I/O modules.
    • Inputs:
      • Push buttons for open, close, stop
      • Selector switch
      • Radar sensor (simulated)
      • Photoelectric sensor simulated using a push button
    • Outputs:
      • LEDs to indicate system states (open, close, fault, maintenance)
      • Stack/tower lights
      • DC motors to visually simulate door movement
  • Ladder Logic Programming
    • Programming Environment: Developed using Studio 5000 with modular ladder logic blocks. Controlled motor states, safety conditions, and lockout behaviors.
    • Motor Control Simulation: Open/close buttons used with seal-in logic to simulate motor activation. Interlocks prevented the system from executing both commands at once. Motors remained off after power loss until reset manually.
    • Cycle Counter Logic: One full open-close sequence = one cycle. After 8 cycles:
      • Disables simulated motor output
      • Red light indicates maintenance required
      • System can only be re-enabled via a manual reset override
    • Emergency Stop Logic: Three emergency stop buttons wired in: two on simulated keypads, one on the remote FOB. Immediately cuts motor output and locks system until reset.
    • Obstruction Detection Simulation: Simulated photoelectric sensor using a push button. When pressed during “closing,” motor logic reversed to simulate the door reopening. Yellow light activated to indicate object detection.
    • Manual Override Switch: Overrides lockout state to allow emergency opening even after cycle limit is reached. Adds a fail-safe in case the system needs to be bypassed.
    • Failsafe Behavior: Seal-in logic ensures the system doesn't auto-restart after power restoration. System state (cycle count, lockout) is preserved until manually reset.
  • Fully functional PLC-based simulation of an automated garage system.
  • Successfully implemented logic for motor control, emergency stops, obstruction detection, and cycle-based lockout.
  • Real components (buttons, motors, LEDs) provided visual and functional feedback.
  • Simulated photoelectric sensor with a push button worked effectively for obstruction logic.
  • Reinforced understanding of ladder logic, safety design, and real-world PLC systems — even in a controlled simulation environment.
